# TODO API 設計書

## 概要

TODOアプリケーションのREST API設計書です。
空欄を埋めて、API設計書を完成させてください。

## ベースURL

```
http://localhost:8000
```

## エンドポイント一覧

| 操作     | HTTPメソッド | エンドポイント | 説明                     |
| -------- | ------------ | -------------- | ------------------------ |
| 一覧取得 | **\_\_\_**   | **\_\_\_**     | 全TODOを取得             |
| 新規追加 | **\_\_\_**   | **\_\_\_**     | 新しいTODOを追加         |
| 完了切替 | **\_\_\_**   | **\_\_\_**     | 指定IDのTODOのdoneを更新 |
| 削除     | **\_\_\_**   | **\_\_\_**     | 指定IDのTODOを削除       |

## 各APIの詳細

### 1. TODO一覧取得

- **メソッド:** **\_\_\_**
- **エンドポイント:** **\_\_\_**
- **リクエストボディ:** なし
- **レスポンス例:**

```json
[
  { "id": 1, "title": "買い物に行く", "done": false },
  { "id": 2, "title": "レポートを書く", "done": true }
]
```

### 2. TODO新規追加

- **メソッド:** **\_\_\_**
- **エンドポイント:** **\_\_\_**
- **リクエストボディ:**

```json
{
  "title": "_______"
}
```

- **レスポンス例:**

```json
{
  "id": ___,
  "title": "_______",
  "done": _______
}
```

### 3. TODO完了切替

- **メソッド:** **\_\_\_**
- **エンドポイント:** **\_\_\_**
- **リクエストボディ:**

```json
{
  "done": _______
}
```

- **レスポンス例:**

```json
{
  "id": ___,
  "title": "_______",
  "done": _______
}
```

### 4. TODO削除

- **メソッド:** **\_\_\_**
- **エンドポイント:** **\_\_\_**
- **リクエストボディ:** **\_\_\_**
- **レスポンス例:**

```json
{
  "message": "_______",
  "id": ___
}
```

## データモデル

### TODOオブジェクト

| フィールド | 型         | 説明                       |
| ---------- | ---------- | -------------------------- |
| id         | **\_\_\_** | TODOの一意なID（自動採番） |
| title      | **\_\_\_** | TODOのタイトル             |
| done       | **\_\_\_** | 完了状態（true/false）     |

## エラーレスポンス

存在しないIDを指定した場合:

- **ステータスコード:** **\_\_\_**
- **レスポンス:**

```json
{
  "detail": "_______"
}
```
